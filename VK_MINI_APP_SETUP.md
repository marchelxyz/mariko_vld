## 🔑 Краткая памятка по ключам

**Что нужно знать:**
- **ID приложения** — не секретный, можно использовать везде (находится в настройках приложения)
- **Секретный ключ (Service Key)** — НЕ нужен для фронтенда! Только для сервера (если используете VK API на бэкенде)
- **Доверенные домены** — нужно указать в настройках ВК (например: `vhachapuri.ru`)

**Для работы мини-приложения достаточно:**
1. Указать URL приложения в настройках ВК
2. Указать доверенные домены
3. Всё! Секретные ключи не требуются для базовой работы

1. Нажмите кнопку **"Создать приложение"**
2. Выберите тип приложения: **"Мини-приложение"**

   - **Название приложения**: Хачапури Марико
   - **Описание**: Приложение для заказа еды из ресторанов сети "Хачапури Марико"
   - **Категория**: Еда и напитки
   - **Возрастные ограничения**: 18+ (если требуется)

## Шаг 3: Настройка мини-приложения

После создания приложения перейдите в раздел **"Настройки"** → **"Мини-приложение"**:

### 3.1. Основные настройки

- **URL приложения**: `https://vhachapuri.ru` (или ваш домен, где размещен фронтенд)
- **URL для разработки**: `http://localhost:8080` (для локальной разработки)
- **Разрешения (Scope)**: 
  - `friends` - доступ к списку друзей
  - `email` - доступ к email пользователя (если нужен)
  - `phone_number` - доступ к номеру телефона (если нужен)

### 3.2. Безопасность

#### Секретный ключ (Service Key)

**⚠️ ВАЖНО: Для базовой работы мини-приложения секретный ключ НЕ нужен!**

Он нужен только если вы планируете:
- Делать серверные запросы к VK API
- Проверять подпись запросов от ВК на бэкенде

**Где найти (если всё-таки нужен):**
1. Зайдите на [dev.vk.com](https://dev.vk.com)
2. Выберите ваше приложение
3. Перейдите в раздел **"Настройки"** (в левом меню)
4. Найдите вкладку **"Работа с API"** или **"API"**
5. Найдите поле **"Сервисный ключ доступа"** (Service Key) — это длинная строка символов
6. Скопируйте его (но помните: используйте только на сервере!)

**Для чего нужен:**
- Для проверки подписи запросов от ВК на сервере (если вы используете VK API на бэкенде)
- Для серверных запросов к VK API без ограничений по частоте

**Где использовать:**
- ⚠️ **НЕ вставляйте секретный ключ в фронтенд код!** Это небезопасно.
- Используйте его только на сервере (бэкенд) в переменных окружения
- Добавьте в `.env` файл бэкенда (если планируете использовать VK API на сервере):

```env
# В backend/server/.env (если нужно)
VK_SERVICE_KEY=ваш_сервисный_ключ_здесь
```

**Важно:** Для работы мини-приложения на фронтенде секретный ключ **НЕ требуется**. Он нужен только если вы планируете делать серверные запросы к VK API.

#### Доверенные домены

**✅ Это ОБЯЗАТЕЛЬНО нужно настроить!**

**Где указать:**
1. Зайдите на [dev.vk.com](https://dev.vk.com)
2. Выберите ваше приложение
3. Перейдите в раздел **"Настройки"** (в левом меню)
4. Найдите вкладку **"Мини-приложение"** или **"Mini Apps"**
5. Прокрутите вниз до раздела **"Доверенные домены"** или **"Allowed domains"**
6. В поле ввода добавьте домены (каждый с новой строки):
   ```
   vhachapuri.ru
   *.vhachapuri.ru
   ```
7. Нажмите **"Сохранить"** или **"Применить"**

**Важно:** Без указания доменов приложение не будет работать! ВК блокирует загрузку с неразрешенных доменов.

**Для чего нужны:**
- ВКонтакте разрешает загрузку мини-приложения только с указанных доменов
- Это защита от подделки и фишинга
- Без указания домена приложение не будет работать

**Пример заполнения:**
```
vhachapuri.ru
*.vhachapuri.ru
```

Или если у вас есть конкретные поддомены:
```
vhachapuri.ru
app.vhachapuri.ru
www.vhachapuri.ru
```

### 3.3. Платформа и версии

- **Платформа**: Web
- **Минимальная версия API**: 5.131 (рекомендуется последняя стабильная)

## Шаг 6: Использование VK контекста в компонентах

Теперь вы можете использовать VK контекст в любом компоненте:

```typescript
import { useVK } from '@/contexts';

function MyComponent() {
  const { isVK, user, vkBridge } = useVK();

  if (isVK && user) {
    return (
      <div>
        <p>Привет, {user.first_name} {user.last_name}!</p>
        {user.photo_100 && (
          <img src={user.photo_100} alt="Аватар" />
        )}
      </div>
    );
  }

  return <div>Приложение работает вне ВКонтакте</div>;
}
```

### Пример: Отправка события в ВК

```typescript
import { useVK } from '@/contexts';

function OrderButton() {
  const { vkBridge } = useVK();

  const handleOrder = async () => {
    // Отправка события в ВК для аналитики
    if (vkBridge) {
      await vkBridge.send('VKWebAppEvent', {
        event_name: 'order_placed',
        event_params: {
          order_id: '12345',
          total: 1500
        }
      });
    }
    
    // Ваша логика оформления заказа
  };

  return <button onClick={handleOrder}>Оформить заказ</button>;
}
```

   - Используйте [VK Mini Apps Dev Tools](https://dev.vk.com/mini-apps/dev-tools) на сайте ВК

3. **Вариант 3: Прямое тестирование**

## ❓ Часто задаваемые вопросы (FAQ)

### Вопрос: Нужен ли мне секретный ключ для работы мини-приложения?

**Ответ:** Нет! Для базовой работы мини-приложения секретный ключ **НЕ нужен**. 

Секретный ключ нужен только если:
- Вы делаете серверные запросы к VK API
- Вы проверяете подпись запросов от ВК на бэкенде

Для работы через VK Bridge (что и используется в этом проекте) секретный ключ не требуется.

### Вопрос: Где найти ID приложения?

**Ответ:** 
1. Зайдите на [dev.vk.com](https://dev.vk.com)
2. Выберите ваше приложение
3. ID будет виден в правом верхнем углу или в разделе "Настройки" → "Основная информация"
4. Это число, например: `12345678`

### Вопрос: Куда вставлять секретный ключ, если он всё-таки нужен?

**Ответ:** 
- ⚠️ **НИКОГДА не вставляйте секретный ключ в фронтенд код!**
- Используйте его только на сервере (бэкенд)
- Добавьте в `backend/server/.env`:
  ```env
  VK_SERVICE_KEY=ваш_ключ_здесь
  ```

### Вопрос: Что такое "доверенные домены" и где их указать?

**Ответ:** 
Доверенные домены — это список доменов, с которых ВК разрешает загружать ваше мини-приложение.

**Где указать:**
1. [dev.vk.com](https://dev.vk.com) → Ваше приложение → Настройки → Мини-приложение
2. Найдите раздел "Доверенные домены"
3. Добавьте домены (каждый с новой строки):
   ```
   vhachapuri.ru
   *.vhachapuri.ru
   ```

**Важно:** Без указания доменов приложение не будет работать!

### Вопрос: Можно ли использовать localhost для разработки?

**Ответ:** Да! В настройках приложения укажите:
- **URL для разработки**: `http://localhost:8080`
- В доверенных доменах localhost обычно не требуется, но можно добавить `localhost` для надежности

### Вопрос: Приложение не открывается в ВК, что делать?

**Ответ:** Проверьте:
1. ✅ Указан ли URL приложения в настройках ВК
2. ✅ Добавлены ли домены в "Доверенные домены"
3. ✅ Используется ли HTTPS для продакшена (обязательно!)
4. ✅ Открыта ли консоль браузера (F12) — там могут быть ошибки
5. ✅ Правильно ли указан URL (должен быть полный: `https://vhachapuri.ru`, а не `vhachapuri.ru`)

### Вопрос: Нужно ли что-то менять в коде после настройки на портале ВК?

**Ответ:** Нет! Код уже готов. Просто:
1. Настройте приложение на портале ВК
2. Укажите URL и домены
3. Задеплойте фронтенд
4. Всё должно работать!

## Дополнительные настройки

### Настройка темы

ВК мини-приложения поддерживают светлую и темную темы. Можно использовать:

```typescript
const theme = await vk.Bridge.send('VKWebAppGetTheme');
// theme.scheme будет 'bright' или 'dark'
```

Если нужно интегрировать платежи ВК:

1. Включите платежи в настройках приложения
2. Используйте `VKWebAppOpenPayForm` для открытия формы оплаты
3. Обрабатывайте события платежей через `VKWebAppPaymentResult`

### Уведомления

Для отправки уведомлений пользователям:

1. Включите уведомления в настройках приложения

3. **Проверка подписи**: Всегда проверяйте подпись запросов от ВК на сервере
