# Простой способ создать PR

## Проблема с вводом команды

Если при вводе команды `.\create_pr_fixed.ps1` PowerShell обрезает команду или не находит файл, используйте один из способов ниже.

## Решение 1: Использовать простой скрипт-обертку

Выполните:
```powershell
.\run_create_pr.ps1
```

Этот скрипт автоматически найдет и запустит `create_pr_fixed.ps1`.

## Решение 2: Использовать полный путь через переменную

```powershell
& "$PWD\create_pr_fixed.ps1"
```

## Решение 3: Использовать Invoke-Expression

```powershell
Invoke-Expression "& '$PWD\create_pr_fixed.ps1'"
```

## Решение 4: Использовать Git Bash (Самое надежное)

**Как открыть Git Bash:**
- **Способ 1:** Правой кнопкой мыши в папке проекта → "Git Bash Here"
- **Способ 2:** Меню Пуск → найдите "Git Bash"
- **Способ 3:** Настройте Git Bash как терминал по умолчанию в Cursor

**Подробная инструкция:** См. файл `GIT_BASH_SETUP.md`

После открытия Git Bash выполните:
```bash
./create_pr.sh
```

## Решение 5: Вручную через Git Bash

Если все скрипты не работают, выполните команды вручную в Git Bash:

```bash
# Перейти в директорию проекта
cd "/c/Users/Роман/OneDrive/Desktop/Договора НПД Мазов/03/Проект Марико апп/Код для курсора/mariko_vld"

# Создать ветку
git checkout -b feat/cart-item-quantity-limit

# Добавить изменения
git add .

# Создать коммит
git commit -m "feat: добавлено ограничение максимального количества одинаковых блюд в корзине

- Добавлено поле max_cart_item_quantity в таблицу restaurants (по умолчанию 10)
- Добавлена миграция БД для нового поля
- Обновлены API для создания/обновления ресторанов
- Добавлена настройка maxCartItemQuantity в админ панели
- Реализована проверка лимита при добавлении блюд в корзину
- Добавлены уведомления при достижении лимита
- Добавлена поддержка maxCartItemQuantity в MenuItemComponent
- Кнопка + становится неактивной при достижении максимального количества"

# Отправить изменения
git push origin feat/cart-item-quantity-limit

# Создать PR
gh pr create --base main --head feat/cart-item-quantity-limit --title "feat: ограничение максимального количества одинаковых блюд в корзине" --body-file PR_DESCRIPTION.md --web
```

## Почему Git Bash лучше?

✅ Нет проблем с кодировкой  
✅ Корректно работает с кириллицей в путях  
✅ Стабильная работа со скриптами  
✅ Не нужно настраивать профиль

## Рекомендация

**Используйте Git Bash** - это самое надежное решение для работы с git в Windows, особенно когда в путях есть кириллица.
