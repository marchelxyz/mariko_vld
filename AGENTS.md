# AGENTS.md

Правила работы для всех последующих сессий в этом репозитории.

## 1) Коммиты
- Все новые commit message писать только на русском языке.
- Рекомендуемый формат: `type(scope): описание на русском`.
- Пример: `fix(menu): исправлено отображение недоступных блюд`.

## 2) Работа с проблемами
- Перед началом любой диагностики сначала открывать `TROUBLESHOOTING.md`.
- Сначала искать совпадающие симптомы и применять уже проверенное решение.
- Если проблема новая, обязательно добавить её в `TROUBLESHOOTING.md`:
  - дата;
  - симптомы;
  - причина;
  - решение;
  - проверка;
  - связанный commit (если есть).

## 3) Поддержка базы знаний
- При каждом изменении `TROUBLESHOOTING.md` обновлять поле "Последнее обновление".
- Не удалять старые кейсы без явной причины, чтобы сохранять историю инцидентов.

## 4) Ограничение по VK версии
- До отдельного явного запроса от владельца проекта не вносить изменения, связанные с VK-версией приложения.
- Любые правки VK (UX, API, авторизация, маршруты, SDK, окружение) выполнять только после прямого подтверждения в текущем диалоге.

## 5) Правило деплоя (test/prod)
- Перед деплоем обязательно сверять, к какому GitHub-репозиторию привязан целевой сервер в TimeWeb.
- Если деплой в **prod (Mini App Хачапури Марико)**: пушить в репозиторий `prodrepo` (`marchelxyz/mariko_vld`) и выполнять деплой оттуда.
- Если деплой в **test**: пушить только в `origin` (`ineedaglokk/MARIKOTEST`) и **не трогать prod** без отдельного явного запроса.
- Не смешивать test/prod деплой в одном действии без прямого подтверждения владельца.
