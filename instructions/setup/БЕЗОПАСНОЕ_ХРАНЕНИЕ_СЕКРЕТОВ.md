# 🔐 Безопасное хранение секретов и API ключей

**Критически важно:** API ключи, токены и пароли НИКОГДА не должны попадать в git, чаты или публичные места!

---

## 📋 **БЫСТРЫЙ СТАРТ**

### **1. Создать файл для секретов:**
```bash
# Скопировать шаблон
cp .env.secrets.example .env.secrets

# Открыть и заполнить реальными данными
nano .env.secrets
```

### **2. Проверить что файл в .gitignore:**
```bash
# Должно вернуть: .env.secrets
grep "\.env\.secrets" .gitignore
```

### **3. Убедиться что файл НЕ в git:**
```bash
# Не должно показать .env.secrets
git status
```

---

## ⚠️ **ЧТО НЕЛЬЗЯ ДЕЛАТЬ**

### **❌ НИКОГДА:**

1. **Не коммитить секреты в git**
   ```bash
   # ПЛОХО:
   git add .env.secrets
   git commit -m "add secrets"
   ```

2. **Не публиковать в чатах/форумах**
   ```
   # ПЛОХО:
   "Вот мой API ключ: YOUR_IIKO_API_LOGIN"
   ```

3. **Не хранить в публичных репозиториях**
   - GitHub public repos
   - Gists
   - Pastebin

4. **Не хардкодить в коде**
   ```javascript
   // ПЛОХО:
   const API_KEY = "YOUR_IIKO_API_LOGIN";

   // ХОРОШО:
   const API_KEY = process.env.IIKO_API_LOGIN;
   ```

5. **Не отправлять по email без шифрования**

6. **Не хранить в незашифрованных облаках**
   - Google Drive
   - Dropbox (без шифрования)

## ✅ **КАК ПРАВИЛЬНО ХРАНИТЬ**

### **Для локальной разработки:**

#### **Вариант 1: Файл `.env.secrets` (рекомендуется)**

```bash
# 1. Создать файл
cp .env.secrets.example .env.secrets

# 2. Заполнить реальными данными
nano .env.secrets

# 3. Использовать в коде
# Автоматически загружается через dotenv
```

**Структура:**
```
.env.secrets          # Реальные секреты (в .gitignore)
.env.secrets.example  # Шаблон без секретов (в git)
```

#### **Вариант 2: Environment Variables**

```bash
# Добавить в ~/.bashrc или ~/.zshrc
export IIKO_PROD_API_LOGIN="ваш-ключ"
export YOOKASSA_PROD_SECRET_KEY="ваш-секрет"
```

---

### **Для production сервера:**

#### **Вариант 1: Timeweb App Platform ENV**

```bash
# Через Timeweb API
curl -X PATCH "https://api.timeweb.cloud/api/v1/app-presets/154595" \
  -H "Authorization: Bearer YOUR_TIMEWEB_TOKEN" \
  -d '{
    "env_version": {
      "vars": {
        "IIKO_PROD_API_LOGIN": "ваш-ключ",
        "DATABASE_URL": "ваш-database-url"
      }
    }
  }'
```

#### **Вариант 2: Серверные переменные окружения**

```bash
# На сервере создать /root/.env.secrets
ssh root@server
nano /root/.env.secrets

# Загружать перед запуском
export $(cat /root/.env.secrets | xargs)
pm2 restart all
```

---

### **Для CI/CD (GitHub Actions, GitLab CI):**

```yaml
# GitHub Actions: Settings → Secrets
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Deploy
        env:
          IIKO_API_LOGIN: ${{ secrets.IIKO_API_LOGIN }}
          DATABASE_URL: ${{ secrets.DATABASE_URL }}
```

## 🔄 **РОТАЦИЯ КЛЮЧЕЙ (регулярная замена)**

### **Когда менять ключи:**

1. **Немедленно**, если ключ утёк:
   - Показали в чате
   - Закоммитили в git
   - Попал в логи

2. **Регулярно** (каждые 3-6 месяцев):
   - Для критичных сервисов

3. **При смене команды:**
   - Уволился сотрудник с доступом

### **Как сменить ключ:**

#### **Пример: iiko API Login**

```bash
# 1. Создать новый ключ в iikoWeb
# 2. Обновить .env.secrets
nano .env.secrets
# Изменить IIKO_PROD_API_LOGIN на новый

# 3. Перезапустить сервисы
pm2 restart all

# 4. Проверить что всё работает
curl https://your-backend.ru/api/health

# 5. ТОЛЬКО ПОСЛЕ ПРОВЕРКИ: отозвать старый ключ в iikoWeb
```

## 🚨 **ЧТО ДЕЛАТЬ ЕСЛИ КЛЮЧ УТЁК**

### **Чеклист действий (СРОЧНО!):**

#### **1. Немедленно отозвать ключ:**

**iiko:**
- Зайти в iikoWeb → Внешние заказы → Настройки Cloud API
- Удалить утёкший ключ
- Создать новый

**ЮКасса:**
- Личный кабинет → API → Отозвать секретный ключ
- Сгенерировать новый

**Telegram Bot:**
- @BotFather → /revoke → создать новый токен

#### **2. Обновить везде новый ключ:**
```bash
# Локально
nano .env.secrets

# На сервере
ssh root@server
nano /root/.env.secrets

# Timeweb App Platform
# Обновить через API или панель управления

# Перезапустить
pm2 restart all
```

#### **3. Проверить логи на подозрительную активность:**
```bash
# Проверить заказы в iiko
# Проверить платежи в ЮКассе
# Проверить логи сервера
```

#### **4. Удалить ключ из git истории (если закоммитили):**
```bash
# ОПАСНО! Меняет историю git
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch .env.secrets" \
  --prune-empty --tag-name-filter cat -- --all

# Форс-пуш (ОСТОРОЖНО!)
git push origin --force --all
```

## 🔧 **КАК ИСПОЛЬЗОВАТЬ В КОДЕ**

### **Node.js (backend):**

```javascript
import dotenv from 'dotenv';
import fs from 'fs';

// Загрузить .env
dotenv.config();

// Загрузить .env.secrets (если существует)
const secretsPath = '.env.secrets';
if (fs.existsSync(secretsPath)) {
  dotenv.config({ path: secretsPath, override: false });
}

// Использовать
const apiLogin = process.env.IIKO_PROD_API_LOGIN;
const secretKey = process.env.YOOKASSA_PROD_SECRET_KEY;

// НИКОГДА не логировать!
console.log('API Login:', apiLogin);  // ❌ ПЛОХО
console.log('API Login:', '***');     // ✅ ХОРОШО
```
