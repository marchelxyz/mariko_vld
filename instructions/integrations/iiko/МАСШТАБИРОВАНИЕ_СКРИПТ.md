# 🚀 МАСШТАБИРОВАНИЕ: Подключение нескольких ресторанов к iiko

**Краткая инструкция:** Что написать и куда нажать

---

## ❓ Что это делает?

Вы пишете список ресторанов в один JSON файл, запускаете одну команду, и **все рестораны автоматически подключаются к iiko** за 30 минут.

**Без скрипта:** 2-4 часа на каждый ресторан вручную
**Со скриптом:** 30 минут на все рестораны сразу

---

## 📋 ШАГ 1: Заполнить список ресторанов

### 1.1 Скопировать шаблон
```bash
cp documentation/templates/iiko-network-restaurants.example.json \
   documentation/templates/iiko-network-restaurants.prod.json
```

### 1.2 Открыть файл
```bash
nano documentation/templates/iiko-network-restaurants.prod.json
```

### 1.3 Заполнить для ваших ресторанов

**Пример для 3 ресторанов:**

```json
{
  "restaurants": [
    {
      "restaurantId": "nn-rozh",
      "apiLogin": "YOUR_IIKO_API_LOGIN",
      "organizationId": "2e688113-401c-4d48-b28c-53852ece72aa",
      "terminalGroupId": "dd70ac26-e6c9-9baf-019b-c1bdd21a0066",
      "sourceKey": "mariko-main"
    },
    {
      "restaurantId": "nn-park",
      "apiLogin": "ВАШ_API_LOGIN_ДЛЯ_ПАРКА",
      "organizationId": "ВАШ_ORG_ID_ДЛЯ_ПАРКА",
      "terminalGroupId": "ВАШ_TERMINAL_ID_ДЛЯ_ПАРКА",
      "sourceKey": "mariko-main"
    },
    {
      "restaurantId": "spb-sadovaya",
      "apiLogin": "ВАШ_API_LOGIN_ДЛЯ_СПБ",
      "organizationId": "ВАШ_ORG_ID_ДЛЯ_СПБ",
      "terminalGroupId": "ВАШ_TERMINAL_ID_ДЛЯ_СПБ",
      "sourceKey": "mariko-main"
    }
  ]
}
```

**Где взять данные для каждого ресторана?**
- `apiLogin` - из iiko Cloud API (создать в настройках)
- `organizationId` - из URL iikoWeb (после `organizationId=`)
- `terminalGroupId` - через endpoint или из настроек терминалов

**Сохранить:** `Ctrl+O`, `Enter`, `Ctrl+X`

---

## 🔧 ШАГ 2: Настроить переменные

```bash
export BACKEND_URL=https://apps.vhachapuri.ru
export ADMIN_TELEGRAM_ID=577222108
```

**Замените:**
- `https://apps.vhachapuri.ru` - на ваш домен
- `577222108` - на ваш Telegram ID

---

## ✅ ШАГ 3: Запустить скрипт

### 3.1 Проверка (без изменений)

```bash
node scripts/iiko/onboard-network.mjs \
  --file documentation/templates/iiko-network-restaurants.prod.json \
  --backend-url "$BACKEND_URL" \
  --admin-telegram-id "$ADMIN_TELEGRAM_ID" \
  --skip-menu-sync
```

**Что будет:**
- ✅ Проверит доступность iiko API для каждого ресторана
- ✅ Выведет терминальные группы и типы оплаты
- ❌ **НЕ** изменит меню и конфигурацию

**Если всё ОК → переходим к следующему шагу**

---

### 3.2 Подключение (реальное)

```bash
node scripts/iiko/onboard-network.mjs \
  --file documentation/templates/iiko-network-restaurants.prod.json \
  --backend-url "$BACKEND_URL" \
  --admin-telegram-id "$ADMIN_TELEGRAM_ID" \
  --apply-menu-sync \
  --report-file documentation/reports/iiko-rollout-$(date +%Y%m%d).json
```

**Что будет:**
- ✅ Добавит конфигурацию iiko для каждого ресторана в БД
- ✅ Синхронизирует меню из iiko (добавит `iiko_product_id` для всех блюд)
- ✅ Сохранит подробный отчёт в `documentation/reports/`

## 🧪 ШАГ 5: Протестировать

Создайте тестовый заказ для каждого ресторана через Telegram Mini App:

1. **Самовывоз + наличные** → должен попасть в iiko
2. **Доставка + онлайн** → оплата + заказ в iiko

**Проверить статус:**
```bash
curl "https://apps.vhachapuri.ru/api/db/recent-orders?key=mariko-iiko-setup-2024&limit=5"
```

Ищите:
- `provider_status: "sent"` - заказ отправлен в iiko ✅
- `provider_order_id: "..."` - ID заказа в iiko ✅

## 📞 Быстрая помощь

**Ошибка при запуске?**
```bash
# Проверить файл конфигурации
cat documentation/templates/iiko-network-restaurants.prod.json

# Проверить доступность backend
curl "$BACKEND_URL/health"

# Проверить доступ к iiko API
curl "https://api-ru.iiko.services/api/1/access_token" \
  -d "apiLogin=ВАШ_API_LOGIN"
```

**Скрипт не находит рестораны?**
- Убедитесь, что `restaurantId` в JSON совпадают с ID в базе данных
- Проверьте: `SELECT id FROM restaurants`
